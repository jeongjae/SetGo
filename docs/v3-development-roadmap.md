# SetGo v3 Development Roadmap

Canonical v3 IA update: see `docs/v3-strong-hevy-menu-migration.md` for the current Strong/Hevy-style menu migration plan. The v3 bottom navigation target is `Today / Routines / History / Insights / More`.

SetGo v2의 개발 진척 상황을 진단하고, 핵심 벤치마킹 대상(Strong, Hevy, Fitbod)과의 격차를 분석하여 향후 SetGo의 다음 단계인 v3의 개발 목표와 구체적인 개발 계획을 수립합니다.

---

### 🔍 주요 성과 (Key Achievements)
1. **HIG 준수 모바일 최적화**: 화면 겹침 및 불필요한 레이블을 정리하고, 한 손 조작이 용이한 세그먼트 컨트롤 및 간결한 로우 레이아웃을 도입했습니다.
2. **도메인 중심의 로컬 추천 구현**: 클라우드나 AI API에 의존하지 않고, 브라우저 로컬 데이터(IndexedDB)만으로 신뢰도 높은 점진적 과부하 및 루틴 회복 추천 엔진을 완성했습니다.
3. **러닝/자유 운동 분리**: 오늘 화면에서 러닝/자유 세그먼트 선택 시 복잡한 루틴 정보가 노출되지 않도록 화면 구조를 깔끔하게 분리했습니다.

## 2. 벤치마킹 앱 대비 격차 분석 (Benchmark Gap Analysis)

현재의 SetGo는 로깅 속도와 스마트 기본값 영역에서 유의미한 진보를 이루었으나, 상용 앱들과 비교할 때 아직 다음과 같은 기능적/경험적 격차가 존재합니다.

### 📋 벤치마크 대상별 격차 비교

| 비교 영역 | Strong / Hevy (로깅 전문) | Fitbod / Fibod (추천 특화) | SetGo v2 (현재 상태) | SetGo v3 보완 방향 |
| :--- | :--- | :--- | :--- | :--- |
| **로깅 입력 속도** | - 빈 필드에 이전 기록 연동 플레이스홀더 제공<br>- 다음 세트로 포커스 자동 이동 | - 단순 기록 기능 제공 하나 로깅 속도보다는 추천에 집중 | - Tap-to-Copy 및 수치 증감 Segmented Control 지원<br>- 키패드 입력은 가능하나 자동 포커스 이동 약함 | - **키패드 입력 포커스 흐름 개선**<br>- 플레이스홀더를 통한 입력 편의성 강화 |
| **성취 및 피드백** | - PR(최고기록) 달성 시 트로피/폭죽 효과<br>- 운동 직후 총 볼륨 및 1RM 상승 요약 카드 제공 | - 운동 완료 요약 제공 | - 볼륨 및 세션 정보 저장<br>- 단순 수치 위주의 역사 기록 및 요약 제공 | - **PR 시각 피드백 효과**<br>- **운동 후 성취 요약 디자인 강화 (공유 카드 등)** |
| **휴식 타이머** | - 세트 완료 시 자동 카운트다운 시작<br>- 푸시/진동 알림 지원 (백그라운드 포함) | - 세션 흐름에 맞춘 기본 타이머 제공 | - 플로팅 휴식 타이머 제공하나 수동 활성화 위주<br>- iOS PWA 백그라운드 타이머 제한 | - **세트 체크 시 자동 타이머 시작**<br>- **PWA 웹 알림(Notification API)을 통한 진동/소리 알림** |
| **추천 모델** | - 추천 없음 (사용자가 모든 계획 수립) | - **근육군별 피로 상태(Muscle Recovery) 추적**<br>- 피로도에 따라 오늘 할 운동 목록 실시간 자동 생성 | - 루틴 설정 및 이전 RIR 기록 기준 스마트 기본값 제공 (1차 추천) | - **근육 피로도(Recovery) 시각화(Heatmap/Progress Bar)**<br>- **피로 상태에 따른 유동적인 일일 루틴 자동 조정** |

## 3. SetGo v3 개발 목표 (V3 Goals)

SetGo v3의 메인 슬로건: 
> **"기록을 넘어선 동반자: 직관적인 피로도 시각화와 강력한 모바일 성취감 제공"**

SetGo v3는 **'로컬 퍼스트의 한계를 허무는 로깅 편의성'**, **'지능형 피로도 모델 기반 추천의 시각화'**, **'운동 의욕을 고취하는 피드백 제공'**을 3대 축으로 설정합니다.

### 🎯 핵심 3대 개발 목표
1. **성취 피드백 및 요약 경험 고도화 (Motivate)**
   - 운동을 마쳤을 때 볼륨 증가량과 새로 달성한 PR(Personal Record)을 시각적으로 강조해 운동 만족감을 극대화합니다.
   - SNS나 메신저에 깔끔하게 공유할 수 있는 요약 이미지 카드 렌더링 기능을 제공합니다.
2. **근육 피로도(Recovery) 모델 탑재 및 Insights 시각화 (Insights)**
   - 수행된 strength 볼륨 및 세트 수 데이터를 기반으로 타겟 근육군별 피로 수준을 계산합니다.
   - 직관적인 프로그레스 바 또는 신체 맵(Muscle Heatmap) 형태로 시각화하여 다음 운동 부위를 결정할 때 참고할 수 있도록 합니다.
3. **PWA 최적화 및 스마트 휴식 타이머 (PWA UX)**
   - iOS Safari/Home Screen 환경에서 사용성을 극대화하기 위해 세트 완료 체크와 연동된 자동 스마트 휴식 타이머를 지원합니다.
   - PWA에서 지원 가능한 웹 알림을 활용하여 백그라운드 환경에서도 휴식 종료 타이밍을 인지할 수 있도록 합니다.

## 4. v3 세부 개발 계획 및 마일스톤 (V3 Implementation Roadmap)

### 🗓️ Phase 1: 성취 피드백 및 요약 화면 강화 (P0)
*목표: 운동이 끝났을 때 성취감을 부여하고 기록 검토를 쉽게 만든다.*
- [ ] **PR 실시간 탐지 및 시각 피드백**: 세트 입력 완료 시 해당 세트가 최근 1RM/무게 기준 PR을 경신할 경우 로우 내에 트로피 아이콘 및 마이크로 애니메이션 노출.
- [ ] **운동 완료(Summary) 카드 고도화**:
  - 총 볼륨 변화율(지난 세션 대비 +X%), 운동 시간, 세트 수 요약.
  - 근육 부위별 자극 비율 차트(간단한 원형 차트 또는 가로형 바 차트).
- [ ] **클립보드 이미지 복사 / SNS 공유**: 완성된 요약 카드를 모바일에 알맞은 종횡비의 이미지로 생성하여 복사 및 다운로드할 수 있는 기능 추가.

### 🗓️ Phase 2: 지능형 근육 피로도 모델 및 Insights 탭 개편 (P0)
*목표: Fitbod처럼 내 몸의 상태를 시각적으로 확인하고 추천의 근거로 삼는다.*
- [ ] **근육 피로도(Recovery) 연산 엔진 구현**:
  - 운동별 타겟 주동근/협력근 매핑 테이블 구축.
  - 운동 완료 후 자극된 근육군의 피로도를 100%로 설정하고, 일자 경과에 따라 지수 감쇄(Exponential Decay) 모델을 적용해 회복율 계산 (예: 24시간당 33% 회복).
- [ ] **Insights 탭 - Muscle Recovery Dashboard**:
  - 가슴, 등, 하체, 어깨, 팔 등 주요 부위의 현재 회복 상태(%)를 프로그레스 바 혹은 심플 차트로 시각화.
  - 주간 부위별 수행 세트 수 및 누적 볼륨 추이 분석.
- [ ] **피로도 기반 추천 고도화**:
  - 회복율이 낮은(피로도가 높은) 부위의 루틴이 오늘 계획되어 있다면 경고 메시지를 보여주거나, 회복이 완료된 부위의 운동을 대체 제안.

### 🗓️ Phase 3: PWA 사용성 극대화 및 스마트 타이머 (P1)
*목표: 네이티브 앱 부럽지 않은 끊김 없는 PWA 운동 플로우를 구축한다.*
- [ ] **자동 스마트 휴식 타이머**:
  - 세트 체크박스 활성화 시, 설정된 운동별 휴식 시간에 맞춰 자동으로 휴식 타이머 시작.
  - 타이머가 활성화된 동안 기기 화면이 꺼지지 않도록 Screen Wake Lock API 적용(가능한 브라우저 대상).
- [ ] **입력 포커스 흐름 최적화**:
  - 무게 입력 완료 후 엔터/다음 버튼 클릭 시 횟수 필드로, 횟수 필드에서 완료 시 다음 세트 무게 필드로 자동 포커스 이동 설계.
  - 가상 키보드가 올라왔을 때 전체 레이아웃이 찌그러지지 않고 스크롤 영역이 유연하게 유지되도록 CSS 최적화.

### 🗓️ Phase 4: Apple Health 간접 연동 및 백업 유효성 강화 (P2)
*목표: PWA의 한계를 극복하는 열린 데이터 연동성을 제공한다.*
- [ ] **외부 활동 파일 가져오기/보내기**:
  - 애플 피트니스/스트라바 등에서 내보낸 FIT/GPX 또는 표준 CSV 데이터를 업로드해 SetGo의 Cardio 레코드로 자동 변환 및 병합.
- [ ] **PWA 환경에서의 Apple Health 연동 가이드 문서 탑재**:
  - 사용자가 단축어(Shortcuts) 앱이나 서드파티 툴을 활용하여 애플 건강 데이터를 SetGo로 수동/반자동 전송할 수 있는 직관적인 가이드 페이지 개설.
- [ ] **자동 로컬 JSON 백업**:
  - 주기적으로IndexedDB 전체 스냅샷을 브라우저 로컬 저장소(또는 파일 다운로드 형식)에 안전하게 백업 및 복구할 수 있는 유효성 검사 모듈 강화.
