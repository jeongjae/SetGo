# SetGo v3 최종 개발계획

## 목적

이 문서는 기존 v3 로드맵을 전면 재작성한 최종 개발계획이다. 이전 문서에는 깨진 문자열과 현재 구현 상태와 맞지 않는 내용이 섞여 있었으므로, 여기서는 현재 산출물을 기준으로 남은 작업을 다시 정리한다.

SetGo v3의 최종 목표는 단순한 메뉴 이름 변경이 아니다.

> Strong/Hevy처럼 빠르게 기록할 수 있고, Fitbod처럼 회복 상태와 다음 운동 판단을 도와주며, 로컬 우선 PWA로 데이터를 믿고 관리할 수 있는 운동 기록 앱.

현재 코드베이스에는 v3 방향의 핵심 기반이 이미 들어가 있다.

- 하단 탭: `오늘 / 루틴 / 기록 / 인사이트 / 더보기`
- 운동 기록 화면은 별도 집중 플로우로 분리되어 있다.
- 루틴, 기록, 인사이트가 최상위 메뉴가 되었다.
- 세트 입력 행은 준비/일반 토글, 중량 증감, 이전 세트 복사, Hard 토글, PR 칩 중심으로 단순화되었다.
- `WorkoutSet.isHard`가 RIR 추론이 아니라 명시적 값으로 저장된다.
- Dexie 기반 로컬 저장과 PWA shell이 유지된다.

남은 작업의 핵심은 기능을 더 많이 붙이는 것이 아니라, 현재 산출물을 완성된 운동 제품처럼 정리하는 것이다.

## v3 제품 원칙

### 1. 기록은 빠르고 예측 가능해야 한다

운동 중에는 한 손으로 반복 입력하는 상황이 많다.

- 현재 운동명이 계속 분명히 보여야 한다.
- 현재 진행할 세트가 시각적으로 드러나야 한다.
- 입력칸을 눌렀을 때 화면이 불필요하게 움직이면 안 된다.
- 이전 기록, PR, 준비세트, Hard, 삭제, 완료 동작이 설명 없이 이해되어야 한다.
- 자동 스크롤은 사용자의 맥락을 해치지 않을 때만 작동해야 한다.

### 2. 루틴은 설정 화면이 아니라 훈련 템플릿이어야 한다

루틴 탭은 Strong/Hevy의 루틴 관리처럼 보여야 한다.

- 사용자는 먼저 자신의 루틴 목록과 활성 루틴을 봐야 한다.
- 루틴 시작 버튼이 가장 명확해야 한다.
- 편집은 가능하되 시작 동작보다 앞서면 안 된다.
- 운동 라이브러리와 주간/사이클 계획은 보조 영역으로 내려가야 한다.

### 3. 인사이트는 다음 행동을 설명해야 한다

인사이트는 단순 통계가 아니라 다음 질문에 답해야 한다.

- 내가 좋아지고 있는가?
- 어떤 부위가 과부하인가?
- 어떤 부위가 부족한가?
- 오늘 추천 운동은 최근 부하 기준으로 안전한가?
- 다음 주에는 무엇을 유지하거나 조정해야 하는가?

### 4. 성취 피드백은 동기부여가 되어야 한다

PR과 운동 완료 순간은 앱의 감정적 보상 지점이다.

- PR은 발생 순간에 분명히 보여야 한다.
- 운동 완료 후 오늘의 성과가 요약되어야 한다.
- 공유 또는 복사 가능한 요약이 있어야 한다.
- 피드백은 과하게 시끄럽지 않아야 한다.

### 5. 로컬 우선 데이터는 신뢰감을 줘야 한다

SetGo는 백엔드 없이 동작하므로, 데이터 관리가 제품 경험의 일부가 되어야 한다.

- 사용자는 데이터가 로컬에 저장된다는 점을 이해해야 한다.
- 백업과 복원은 안전하고 검증 가능해야 한다.
- Apple Health 같은 외부 데이터는 PWA 제약을 솔직하게 설명해야 한다.

| 미비점 | 영향 |
|---|---|

## Phase 1. 루틴 제품 화면 재구성

**우선순위:** P0
**목표:** 루틴 탭을 설정 화면이 아니라 루틴을 고르고 시작하는 제품 화면으로 만든다.

### 목표 UX

루틴 탭 첫 화면은 다음 순서로 읽혀야 한다.

1. 활성 루틴
2. 루틴 목록
3. 선택한 루틴의 요일/데이
4. 시작 버튼
5. 편집/복제/삭제 같은 보조 액션

### 작업

- 루틴 화면을 명확한 섹션으로 분리
  - 루틴 홈
  - 루틴 상세
  - 운동 라이브러리
  - 주간/사이클 계획
- 루틴 시작 액션을 상단 핵심 CTA로 승격
  - 선택한 데이 시작
  - 다음 예정 데이 시작
- 루틴 변경, 이름 변경, 복제, 삭제를 명확하게 제공
- 운동 라이브러리와 주간/사이클 계획은 보조 탭 또는 접힌 영역으로 이동
- 루틴 없음, 데이 없음, 운동 없음 상태의 empty state 정리

### 완료 기준

- 루틴 탭을 열면 3초 안에 활성 루틴과 시작할 데이를 알 수 있다.
- 루틴 편집 화면에 들어가지 않고도 운동을 시작할 수 있다.
- 운동 추가/정렬/삭제 기능은 유지된다.
- 루틴 탭이 더보기/설정 화면처럼 보이지 않는다.

## Phase 2. 오늘 화면 단순화

**우선순위:** P0
**목표:** 오늘 화면을 다음 운동 시작에 집중시킨다.

### 목표 UX

오늘 화면은 다음 질문에만 빠르게 답한다.

- 지금 무엇을 하면 되는가?
- 이어서 할 운동이 있는가?
- 추천 이유는 무엇인가?
- 최근에는 무엇을 했는가?

### 작업

- 루틴 선택 UI의 비중 축소
- 추천 루틴/러닝/자유 운동을 빠른 시작 후보로 정리
- 최근 루틴 빠른 시작 영역을 더 compact하게 정리
- 계획과 편집에 가까운 요소는 루틴 탭으로 이동
- 회복 모델 이후 추천 사유에 회복 경고 추가
  - 예: "하체 회복이 낮아 상체 루틴을 추천합니다."

### 완료 기준

- 첫 화면 안에 핵심 CTA가 보인다.
- 일반 상태에서 스크롤 없이 시작/계속하기가 가능하다.
- 오늘 화면이 루틴 설정 화면처럼 보이지 않는다.
- 추천 이유는 한 줄 또는 짧은 두 줄로 읽힌다.

## Phase 3. 운동 기록 최종 다듬기

**우선순위:** P0
**목표:** Strong/Hevy 수준의 빠른 기록 사용성을 완성한다.

### 이미 완료된 기반

- 2줄 세트 행
- 일반/준비 토글
- 명시적 Hard 토글
- 이전 기록 옆 복사 아이콘
- 삭제 버튼 2줄 배치
- PR 칩 배치
- 중량 입력 좌우 `- / +`
- `isHard` 저장

### 남은 작업

- 운동별 중량 증감 단위 적용
  - 우선순위:
    1. 루틴 운동 계획의 값
    2. 추후 운동별 기본값
    3. 기본 `2.5kg`
  - 운동 관리 라인에 `중량단위: Xkg` 표시
  - 세트 행의 `- / +`도 같은 값을 사용
- 입력 흐름 정리
  - kg 입력 후 reps 이동
  - reps 입력 후 RIR 이동
  - RIR 완료 후 불필요한 자동 스크롤 금지
- 현재 세트 focus 검증
  - 현재 행 배경 강조
  - 완료 후 다음 미완료 세트가 자연스럽게 보일 때만 보조 스크롤
- 운동 단위의 "이전 세트 전체 복사"가 직관적이면 추가 검토
- 긴 운동명, 큰 중량값, 375px/390px/427px 화면에서 레이아웃 검증

### 완료 기준

- 운동 기록 화면에 가로 overflow가 없다.
- 이미 보이는 입력칸을 눌렀을 때 운동명이 사라지도록 강제 중앙 정렬하지 않는다.
- 표시된 중량단위와 실제 증감 값이 같다.
- Hard 값이 새로고침 후에도 유지되고 기록/인사이트에 반영된다.
- 이전 기록 복사 기능이 설명 없이 이해된다.

## Phase 4. 성취 피드백과 완료 요약

**우선순위:** P0
**목표:** PR과 운동 완료 순간을 제품의 보상 경험으로 만든다.

### 작업

- PR 표시 개선
  - 기존 `kg PR`, `vol PR`, `PR` 유지
  - PR 발생 행에 작은 시각 강조 추가
  - 처음 발생 시 과하지 않은 micro animation 추가
- 운동 완료 요약 화면 또는 모달 추가
  - 총 볼륨
  - 완료 운동 수
  - 완료 세트 수
  - Hard 세트 수
  - PR 수
  - 이전 유사 세션 대비 볼륨 변화
  - 러닝/유산소 요약
- 공유/복사 가능한 텍스트 요약 MVP 추가
- 기록 카드에 PR 수, Hard 세트 수, 볼륨 요약 칩 추가

### 완료 기준

- 운동 완료 후 오늘 잘한 점이 바로 보인다.
- PR은 기록 중에도 인지되지만 방해되지 않는다.
- 운동 요약을 텍스트로 복사할 수 있다.
- 기록 목록에서 의미 있는 세션을 빠르게 구분할 수 있다.

## Phase 5. 회복 모델 MVP

**우선순위:** P0
**목표:** Fitbod 방향의 핵심 차별점을 만든다.

### 모델 방향

로컬 deterministic 모델로 시작한다. 백엔드나 AI는 사용하지 않는다.

각 완료 strength set에 대해:

- 운동을 주요 근육군에 매핑
- 부하 기여도 계산
  - 기본: `weightKg * reps`
  - Hard: `isHard`면 `1.25` 배수
  - PR: 필요 시 `1.15` 배수
  - 준비세트: `0.25` 배수
- 근육군별로 부하를 배분
- 최근 부하를 fatigue로 변환
- 시간 경과에 따라 fatigue를 감소시키고 recovery percent 계산

초기 공식 예시:

```text
fatigueToday = min(100, normalizedRecentLoad)
recoveryPercent = clamp(0, 100, 100 - fatigueAfterDecay)
dailyDecay = 0.35 ~ 0.45
```

상수는 쉽게 튜닝 가능하게 둔다.

### 작업

- 근육군 매핑 유틸 추가
  - 가슴
  - 등
  - 어깨
  - 이두
  - 삼두
  - 하체
  - 코어
  - 유산소/전신
- `domain/recovery.ts` 추가
- 단위 테스트 추가
  - decay
  - Hard multiplier
  - 준비세트 영향 축소
  - 여러 근육군에 걸친 운동
  - 데이터 없음 fallback
- 인사이트에 회복 요약 추가
- 오늘 추천에 사용할 회복 경고 API 추가

### 완료 기준

- 주요 근육군별 회복률이 계산된다.
- 최근 Hard 세션은 회복률을 낮춘다.
- 시간이 지나면 회복률이 오른다.
- 준비세트는 회복률을 과도하게 낮추지 않는다.
- 오늘 화면에서 계획된 루틴에 대한 간단한 회복 경고를 만들 수 있다.

---

## Phase 6. 인사이트 v3 대시보드

**우선순위:** P0
**목표:** 현재 Stats를 실제 의사결정용 Insights로 확장한다.

### 목표 UX

인사이트는 다음 순서로 구성한다.

1. 진행 상황 요약
2. 회복 대시보드
3. 근육군 부하 균형
4. PR/1RM 추세
5. 다음 주 조정 제안

### 작업

- 기존 Stats 내용을 Progress 영역으로 재배치
- Recovery 섹션 추가
  - 근육군별 회복 bar
  - 가장 피로한 부위
  - 가장 회복된 부위
- 주간 부하 균형 추가
  - 근육군별 세트 수
  - 근육군별 볼륨
  - 근육군별 Hard 비율
- 다음 주 가이드 추가
  - 유지
  - 과부하 부위 감소
  - 부족 부위 추가
  - 회복일 제안
- AI 프롬프트 복사는 보조 기능으로 유지

### 완료 기준

- 인사이트가 "다음에 무엇을 조정할지" 답한다.
- 회복과 근육 균형이 앱 안에서 바로 보인다.
- iPhone 화면에서 과밀하지 않다.
- 기존 통계 테스트가 유지된다.

---

## Phase 7. 기록 화면 정리

**우선순위:** P1
**목표:** 기록 탭을 "언제 무엇을 했는지" 빠르게 확인하는 화면으로 만든다.

### 작업

- 필요하면 캘린더/리스트 모드 전환 추가
- 완료 기록과 진행 중/임시 기록을 시각적으로 구분
- 운동 기록 카드 compact화
  - 루틴/데이
  - 볼륨
  - 세트 수
  - Hard 수
  - PR 수
  - 러닝 거리/시간
- 과거 기록 수정 진입을 명확하게 제공
- 삭제 확인에는 날짜와 요약을 포함

### 완료 기준

- 기록 화면이 인사이트와 역할이 겹치지 않는다.
- 과거 운동을 빠르게 찾고 수정할 수 있다.
- 캘린더/리스트 선택 상태가 자연스럽게 유지된다.

## Phase 8. PWA 타이머와 기기 UX

**우선순위:** P1
**목표:** PWA 제약 안에서 운동 중 타이머 경험을 안정화한다.

### 작업

- 휴식 타이머 설정 정리
  - 기본 휴식 시간
  - 운동별 휴식 시간 노출 개선
  - 사운드/진동/알림 가능 여부 표시
- Notification API 흐름 추가
  - 사용자가 켰을 때만 권한 요청
  - 지원 브라우저에서 타이머 종료 알림
- 지원 환경에서 진동 추가
- 지원 환경에서 운동 중 Screen Wake Lock 사용
- iOS 제한은 깨진 기능처럼 보이지 않게 짧게 안내

### 완료 기준

- 세트 완료 후 타이머가 안정적으로 시작된다.
- 타이머는 운동 흐름 안에서 계속 확인 가능하다.
- 지원 브라우저에서는 종료 알림이 동작한다.
- 미지원 브라우저에서는 제한 안내가 명확하다.

## 권장 실행 순서

1. Phase 0: 제품 완성도 부채와 문서 정리
2. Phase 1: 루틴 제품 화면 재구성
3. Phase 2: 오늘 화면 단순화
4. Phase 3: 운동 기록 최종 다듬기
5. Phase 5: 회복 모델 MVP

## v3 최종 완료 정의

SetGo v3는 다음 조건을 만족할 때 완료로 본다.

- 앱 구조가 Strong/Hevy 방향의 최종 IA를 따른다.
- 루틴 탭이 일급 루틴 관리자처럼 동작한다.
- 오늘 화면은 즉시 시작/계속하기에 집중한다.
- 운동 기록은 빠르고 안정적이며 설명 없이 이해된다.
- 인사이트는 회복, 진행, 근육 부하, 다음 행동 제안을 포함한다.
- PR과 운동 완료 피드백이 동기부여를 준다.
- 기록 탭은 과거 운동 검토와 수정에 적합하다.
- 로컬 데이터, 백업, 복원, 가져오기 흐름을 신뢰할 수 있다.
- iPhone 크기 브라우저와 홈 화면 PWA에서 검증된다.
- 문서, 테스트, 빌드, 배포 상태가 실제 산출물과 일치한다.
