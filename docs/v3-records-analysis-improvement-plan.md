# 인사이트 "기록 분석" 개선 계획

## 결론

현재 기록 분석의 헤드라인 지표는 **월요일 시작 고정 캘린더 주(Mon–Sun)** 를 기준으로 삼아, 주 초반에는 데이터가 비어 보이고 주 후반에는 가득 차 보이는 왜곡이 있다. 반면 근육군 분석은 **롤링 28일**, 일간 추세는 **롤링 14일** 을 써서 한 화면 안에서 기준이 뒤섞여 있다.

개선 방향은 상용 앱(Hevy)의 검증된 패턴을 따른다: **현재 일자 기준 롤링 윈도우를 기본**으로 하고, 사용자가 **기간(최근 7일 / 4주 / 3개월)을 선택**하며, **직전 동일 기간과 비교**한다. 근육군 세트는 운동 과학의 표준 단위인 "주당 세트 수" 타깃과 함께 보여준다.

---

## 1. 현재 구현 진단

`src/pages/StatsPage.tsx`의 `buildStats(...)` (약 410줄)에서 기간 경계가 혼재한다.

| 지표 | 코드 | 윈도우 | 문제 |
|---|---|---|---|
| 주간 총 볼륨 / 세트 / Hard / 전주 대비 | `thisWeekStart = startOfWeek(today)` (Mon 시작) | **고정 캘린더 주** | 주 초반 과소·주 후반 과대. 사용자가 지적한 핵심 |
| 8주 추세 | `eightWeekStart = thisWeekStart - 49` | 캘린더 주 버킷 | 기준이 캘린더 주에 고정 |
| 근육군 부하 분석 | `fourWeekStartKey = today - 27` | **롤링 28일** | 헤드라인과 기준 불일치 |
| 일간 추세 | `twoWeekStart = today - 13` | 롤링 14일 | 또 다른 기준 |
| 연속일/경고 | `today - 14` 루프 | 롤링 14일 | — |

- 기간 선택 UI는 **없음**. 분석은 페이지 진입 시 한 번 계산된다 (`stats` 상태).
- `weekOverWeekPct`는 캘린더 주 대 캘린더 주 비교라 주 초반에는 의미가 약하다.

핵심 결함: **"이번 주"라는 단어가 캘린더 주를 의미해서, 오늘 기준 최근 운동량을 보고 싶은 사용자의 멘탈 모델과 어긋난다.**

## 2. 상용 앱 레퍼런스

### Hevy (가장 직접적인 참고)
- **Last 7 days 바디 그래프**: 오늘 기준 **롤링 7일**. "최근에 무엇을 훈련했나 / 일관성"을 보여주는 글랜스용. → 절대 비어 보이지 않음.
- **Body distribution**: 근육군별 세트 수를 **현재 캘린더 주** 단위로 표시하되, 좌우 화살표로 주 단위 이동(week-over-week) 제공.
- **Muscle distribution (고급)**: 기간 셀렉터 **최근 30일 / 3개월 / 1년 / 전체**, **직전 기간 대비 비교**(month-over-month), "최근 30일을 주간으로", "최근 1년을 월간으로" 세그먼트 제공.
- **Monthly Report**: 월간 리포트를 별도 제공.

→ 시사점: Hevy는 **롤링 글랜스 + 캘린더 주 타깃 점검 + 기간 셀렉터 추세**를 의도적으로 분리해서 모두 제공한다.

### 운동 과학(RP/Stronger by Science 등) — 타깃의 단위
- 근비대 부하 타깃은 **"근육당 주당 hard set 수"** 로 표현: MEV(최소 효과 볼륨) 10–12세트, MAV(최대 적응 볼륨) 15–20세트, 상한 ~20세트.
- → 세트 타깃은 본질적으로 **주 단위**가 자연스럽다. 따라서 "주당 세트" 점검은 7일 윈도우로 환산해서 보여주는 것이 타당하다.

## 3. 개선 설계

### 원칙
1. **모든 윈도우는 오늘 기준 롤링으로 통일**한다(혼재 제거).
2. 헤드라인은 **"최근 7일"** 을 기본값으로 한다 → 주 초반 빈 화면 문제 해결.
3. 사용자가 **기간을 전환**할 수 있다: `최근 7일 / 최근 4주(28일) / 최근 3개월(84일)`.
4. 각 기간은 **직전 동일 길이 기간과 비교**한다(예: 최근 7일 vs 그 이전 7일).
5. 근육군 세트는 선택 기간을 **주당 평균 세트로 환산**해 MEV/MAV 타깃 밴드와 함께 표시한다.

### UI 구성(위→아래)
1. **기간 세그먼트 컨트롤**: `7일 · 4주 · 3개월` (기존 `IOSSegmentedControl` 재사용). 선택값은 `localStorage`에 저장.
2. **요약 카드**: 선택 기간의 총 볼륨 · 운동일 · 총 세트 · Hard 세트 비율, 각각 **직전 기간 대비 변화%**.
3. **추세 차트**: 기간에 맞춰 버킷 변경 — 7일=일간 막대, 4주=주간 막대, 3개월=주간 막대.
4. **근육군 균형**: 주당 평균 세트 + MEV/MAV 밴드(부족/적정/과다). 7일 기간이면 실제 주, 그 이상이면 주당 평균.
5. **회복 대시보드**(기존, 오늘 기준 그대로 유지 — 이미 today-anchored).
6. **다음 주 가이드 / 경고**: 선택 기간 데이터를 근거로 재계산.

### 데이터 모델 변경
- `buildStats`를 `buildStats(data, locale, { window })` 형태로 일반화.
- `AnalysisWindow = { days: 7 | 28 | 84, label }` 도입.
- 현재/직전 기간 경계를 단일 헬퍼로 계산:
  - `currentStart = today - (days - 1)`, `currentEnd = today`
  - `previousStart = today - (2*days - 1)`, `previousEnd = today - days`
- 근육군 세트 → 주당 환산: `setsPerWeek = totalSets / (days / 7)`.

## 4. 단계별 작업

### Phase A — 윈도우 통일 + 기간 셀렉터 (P0, 핵심)
- `startOfWeek` 기반 헤드라인을 롤링 윈도우로 교체.
- 기간 세그먼트(`7일/4주/3개월`) 추가, 선택 상태 + localStorage.
- 요약 카드와 전기간 대비 비교를 선택 기간 기준으로 재계산.
- 기존 i18n `statsWeekOverWeek` 등 "주" 고정 문구를 기간 일반 문구로 교체/추가.
- **완료 기준**: 월요일에 열어도 최근 7일 데이터가 보인다. 모든 카드가 동일한 선택 기간을 따른다.

### Phase B — 근육군 균형의 주당 세트 + 타깃 밴드 (P0)
- 근육군 분석을 "주당 평균 세트"로 환산하고 MEV(10)/MAV(20) 밴드 표시.
- 부족/적정/과다 분류를 주당 세트 기준으로 재정의(`statusForSets` 갱신).
- **완료 기준**: 각 근육군이 주당 세트와 타깃 대비 위치로 읽힌다.

### Phase C — 추세 차트 버킷 적응 + 비교 (P1)
- 기간별 버킷(일/주) 전환, 직전 기간 평균선 또는 비교 수치 추가.
- 8주 고정 추세를 선택 기간에 종속되도록 정리.
- **완료 기준**: 기간 전환 시 차트 해상도가 자연스럽게 바뀐다.

### Phase D — 검증 (P0, 각 단계 끝)
- `buildStats` 윈도우 경계 단위 테스트(경계 날짜 포함/제외, 주당 환산, 빈 데이터 fallback).
- `npm run test` / `build` / `test:e2e` / `test:viewport` 통과.

## 7. 구현 완료 (2026-06-24)

Phase A~D 전체 구현 및 검증 완료.

### 결정된 기본값
- 헤드라인 기본 기간: **7일** (세그먼트 `7일 / 4주 / 3개월`, localStorage `setgo.statsWindow`에 저장).

### 변경 사항
- `buildStats(..., windowDays)`로 일반화. 모든 헤드라인/근육군/추세 지표가 **오늘 기준 롤링 윈도우**를 따른다. `startOfWeek` 캘린더 주 로직 제거.
- 현재 기간 = `[today-(N-1) .. today]`, 직전 기간 = 바로 이전 N일. 전기간 대비 비교가 like-for-like.
- 근육군: 기간 세트를 **주당 평균(`setsPerWeek`)** 으로 환산해 MEV(min)/MAV(max) 밴드와 비교. `MuscleBalancePanel`·상세 카드 모두 주당 기준.
- 추세 레일: 7일=일간 막대, 4주/3개월=주간 막대(`trendGranularity`). 주간 버킷은 오늘 기준 롤링 12주에서 선택 기간만큼 슬라이스.
- 인사이트 헤더에 `IOSSegmentedControl` 기간 셀렉터 추가, 카드 상단에 "최근 N일" 캡션, 비교 타일 라벨을 "이전 기간 대비"로 변경.
