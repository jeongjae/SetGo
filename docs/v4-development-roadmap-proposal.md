# SetGo v4 개발 로드맵 및 기능 제안서 (Roadmap & Feature Proposal)

SetGo v3에서 모바일 사용성과 로컬 저장 기능의 안정성이 검증되었습니다. v4에서는 경쟁 피트니스 앱인 **Strong**, **Hevy**, **Fitbod**의 핵심 강점을 로컬 우선(Local-first) 및 프라이버시 친화적인 방향으로 벤치마킹하여 다음 단계의 사용자 경험을 제공하고자 합니다.

본 제안서는 특히 **(1) 개인 맞춤형 다음 루틴 제안**, **(2) 점진적 과부하 세트 제안**, **(3) 다차원 기록 분석**의 세 가지 영역을 중점적으로 다룹니다.

---

## 2. 핵심 기능 제안 (Core Proposals)

### 영역 A. 다음 운동 루틴 제안 (Next Workout Routine Suggestions)
Fitbod의 자동 추천 모델을 네트워크/서버 비용 없이 브라우저 단에서 IndexedDB를 활용해 **로컬 근육 피로도 모델**로 경량화하여 구현합니다.

1. **로컬 근육 회복 상태 엔진 (Local Recovery Engine)**:
   - 각 운동(Exercise) 마스터 데이터에 주동근(Primary Muscles)과 협응근(Secondary Muscles)을 정의합니다. (예: 벤치 프레스 = 가슴[주], 삼두/전면 삼각근[협])
   - 완료된 세트 수와 세트 유형(특히 Hard 세트)에 비례하여 해당 부위의 피로도를 누적시킵니다.
   - 시간 경과에 따라 피로도를 감쇠(가슴: 48시간 완료 회복, 이두: 24시간 완료 회복 등)하여 부위별 **현재 회복도(0~100%)**를 수치화합니다.
2. **하이브리드 근육 회복 시각화 (Hybrid Muscle Recovery Map)**:
   - 전신 일러스트(인체 실루엣 SVG 맵)와 직관적인 수치(%) 표기를 결합한 하이브리드 인터페이스를 제공합니다.
   - 인체 모델 전/후면의 주요 근육군(가슴, 어깨, 등, 이두, 삼두, 대퇴사두, 햄스트링 등)이 회복도에 따라 색상(빨강/노랑/초록)으로 칠해지고, 그 위에 현재 회복율(예: "가슴 95%") 배지가 직접 표시되어 직관적인 피드백을 제공합니다.
3. **오늘의 추천 루틴 카드 (Today's Recommendation Card)**:
   - "Today" 탭 상단에 오늘 수행하기에 회복 상태가 가장 우수한 루틴을 1~2개 우선순위로 제안합니다.
   - 예: "가슴과 삼두가 95% 회복되었습니다. 오늘 **[루틴 A: Push Day]**를 진행하는 것을 추천합니다."
   - 오버트레이닝 방지: 특정 부위의 회복도가 30% 이하인 상태에서 해당 루틴을 시작하려고 하면 "근육 피로도가 높은 상태입니다"라는 가벼운 경고 팁을 보여줍니다.

### 영역 B. 운동 세트 및 수치 제안 (Progressive Overload Set Prescriptions)
단순 복사를 넘어 부상 위험을 줄이고 체계적으로 강도를 올릴 수 있도록 돕는 점진적 과부하 계산기입니다.

1. **추정 1RM (One-Rep Max) 자동 계산**:
   - 세트 기입 시 Epley 공식을 통해 실시간 추정 1RM을 계산합니다.
     $$\text{Estimated 1RM} = \text{Weight} \times \left(1 + \frac{\text{Reps}}{30}\right)$$
2. **과부하 제안 플레이스홀더 자동 입력 (Placeholder Auto-Prefill)**:
   - 이전 세션 기록 기반 점진적 과부하 제안값(Linear/Double Progression 계산값)을 세트 인풋 박스에 **연한 회색 Placeholder(가입력 값)**로 표시합니다.
   - 사용자가 해당 입력 칸을 탭(포커스)하면 **제안값이 자동으로 실제 입력 값으로 채워지며(Prefill)**, 수동 타이핑 없이 즉시 체크 버튼이나 키보드 엔터를 통해 완료할 수 있어 초고속 기록 환경을 구현합니다.
3. **스마트 웜업 세트 빌더 (Warm-up Set Generator)**:
   - 본 세트의 중량을 기반으로 웜업 단계를 터치 한 번으로 생성합니다.
     - 1세트: 본 세트의 50% x 8 reps
     - 2세트: 본 세트의 70% x 5 reps
     - 3세트: 본 세트의 90% x 2 reps
   - 웜업 세트를 수동으로 일일이 계산해서 추가할 필요가 없어져 세깅 속도가 급격히 향상됩니다.

### 영역 C. 다차원 기록 분석 (Multi-Dimensional Local Analysis)
Strong과 Hevy의 세련된 차트 경험을 SetGo만의 HSL 슬림 다크/글래스모피즘 테마로 차용하여 제공합니다.

1. **부위별 주간 세트 수 달성도 (Weekly Sets by Muscle Group)**:
   - 건강한 근성장을 위해 권장되는 주당 부위별 목표 세트수(예: 가슴 10~20세트) 대비 현재 진행 상황을 원형 게이지(Progress Ring) 또는 수평 막대 차트로 렌더링합니다.
   - 타겟 부위가 부족하거나 과도할 경우 색상 톤(정상: Teal, 부족: Gray, 과도: Danger)을 다르게 표시하여 훈련 균형을 잡도록 돕습니다.
2. **추정 1RM 및 볼륨 트렌드 차트**:
   - 특정 운동을 선택했을 때 기간별(4주/8주/12주/전체) 추정 1RM 변화 곡선과 세션별 총 볼륨(`중량 x 횟수`)을 한데 겹쳐 그리는 이중 축 선그래프를 지원합니다.
3. **피로도 피드백 (Deload Recommendation)**:
   - 최근 4주 대비 이번 주의 세트 수와 Hard 세트 비율을 비교 계산하여 피로도 지표를 추출하고, 피로도가 극도에 달했을 때 "이번 주는 가벼운 중량으로 수행하는 디로드(Deload) 주간을 가져보는 것은 어떨까요?"라는 맞춤형 로컬 리포트를 제공합니다.

## 3. 기술적 설계 (Technical Design - Local-first)

- **상태 관리 및 연산 성능**:
  - 복잡한 회복도나 1RM 계산은 데이터량이 늘어나더라도 버벅임이 없도록 Dexie(IndexedDB) 쿼리를 최적화하고, 계산 결과를 메모리 단에서 `useMemo`로 바인딩하여 컴포넌트 렌더링 지연을 방지합니다.
- **차트 라이브러리**:
  - PWA 번들 사이즈 최소화를 위해 Chart.js나 Recharts 같은 무거운 외부 라이브러리는 배제하고, 리액트 컴포넌트로 구현된 **경량 정적 SVG(Static SVG)**를 활용해 1RM 및 볼륨 추이 그래프를 생성합니다.
  - 확대/축소 및 툴팁 인터랙션을 간소화해 성능과 모바일 가독성을 극대화합니다.
- **데이터 스키마 확장**:
  - `WorkoutSet` 테이블에 실시간 계산된 `estimated1RM` 속성을 저장해 쿼리 성능을 높이고, `ExerciseMaster` 테이블에 주동근/협응근 정보를 포함할 수 있도록 Dexie DB 버전 마이그레이션(v3 -> v4) 설계를 반영합니다.

## 4. 최종 확정된 설계 사양 (Finalized Design Specifications)

1. **가슴, 어깨, 하체 등 전신 SVG 인체 일러스트 맵** 위에 **실시간 회복율(%) 지표**가 오버레이 표시되는 하이브리드 근육 회복 시각화 탑재.
2. 이전 기록 및 1RM을 기반으로 자동 계산된 점진적 과부하 타겟이 **연한 회색 Placeholder**로 제공되고, 탭 즉시 입력 필드에 자동 입력(Prefill)되는 기록 방식 탑재.
3. 통계 페이지에는 PWA의 로딩 성능을 최적화하기 위해 무거운 모듈 없이 직접 렌더링하는 **미니멀 정적 SVG 트렌드 그래프** 탑재.
