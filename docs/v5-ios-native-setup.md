# SetGo v5 iOS Native Setup

## Current Status

SetGo now has a Capacitor iOS scaffold and a SQLite plugin binding path.

- Capacitor app id: `app.setgo.mobile`
- Capacitor app name: `SetGo`
- Web build directory: `dist`
- iOS project: `ios/App/App.xcodeproj`
- SQLite plugin: `@capacitor-community/sqlite`
- SQLite driver binding: `src/storage/capacitorSqliteDriver.ts`
- Repository adapter: `src/storage/nativeSqliteRepository.ts`

The scaffold was created and synced on Windows. Xcode simulator/device verification still requires macOS with Xcode.

## Commands

Install dependencies:

```bash
npm install
```

Build and sync the current web app into the iOS project:

```bash
npm run native:sync
```

Open the iOS project on macOS:

```bash
npm run native:open
```

Then select a simulator or connected iPhone in Xcode and run the app.

## Windows-Only Development Path

When the developer only has a Windows PC, use this split:

```text
Windows PC:
  React / TypeScript / Capacitor config / tests / web deploy

GitHub Actions macOS runner:
  iOS sync / Xcode simulator build

Later, with Apple Developer credentials:
  signed archive / TestFlight upload / iPhone install
```

The workflow `.github/workflows/ios-native-check.yml` builds the iOS simulator app on a macOS runner without code signing. This verifies that the native scaffold, Swift Package setup, Capacitor sync, and SQLite plugin wiring can compile outside the Windows machine.

This does not install the app on a real iPhone and does not upload to TestFlight. Those require Apple Developer signing assets and an App Store Connect setup.

## Migration Fixture

The non-personal migration fixture is `src/storage/nativeMigrationFixture.ts`.

It represents a PWA JSON backup with:

- exercises and routine plans;
- a completed strength workout;
- a completed running/cardio record;
- recommendation snapshots;
- a calendar running override.

`src/storage/nativeMigrationFixture.test.ts` verifies that the fixture previews correctly and imports through the native SQLite repository adapter.

`src/storage/nativeMigration.ts` provides the reusable preview/import layer for native migration. It reports:

- record counts;
- estimated backup size;
- missing relationship errors;
- duplicate imported cardio warnings;
- whether the backup can be imported.

`src/storage/nativeMigrationReceipt.ts` stores a compact migration receipt in Capacitor Preferences after a successful native import. The receipt includes backup/export timestamps, imported counts, estimated size, and warning count.

`src/storage/nativeMigrationRuntime.ts` is the runtime bridge for the future native import UI. It parses JSON backup text, previews it, initializes the native SQLite schema, imports through the native repository, and saves the migration receipt.

## Native Storage Path

The configured SQLite location is:

```text
Library/SetGoDatabase
```

This is intended to preserve workout data across normal app updates. Manual restore should not be part of the normal native update flow.

## Verification Checklist On macOS

1. Run `npm install`.
2. Run `npm run native:sync`.
3. Run `npm run native:open`.
4. In Xcode, build the `App` target for an iPhone simulator.
5. Confirm the app launches and renders the SetGo Today screen.
6. Open More and run `Native Storage Check`.
7. Confirm the first run reports a saved native record.
8. Terminate and relaunch the app.
9. Open More and run `Native Storage Check` again.
10. Confirm the second run reports that a previous run was found.

The probe uses a separate SQLite database named `setgo_probe`. It does not clear or mutate the real SetGo workout database.

## Notes

- `ios/App/App/App/public`, `ios/App/App/capacitor.config.json`, and `ios/App/App/config.xml` are generated by `cap sync` and intentionally ignored.
- `ios/App/CapApp-SPM/Package.swift` is managed by Capacitor and includes the SQLite package.
- If `cap sync ios` is run on Windows, verify local Swift package paths still use `/` separators before building on macOS.
- Do not edit generated Capacitor files unless a native build issue requires it.
